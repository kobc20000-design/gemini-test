# bazi-mcp 분석 문서
## 명리학 공부 + 작명 앱 개선을 위한 참고 자료

# PART 1. bazi-mcp 전반 내용 정리

## 1-1. bazi-mcp란?

- **출처:** Cantian AI (https://github.com/cantian-ai/bazi-mcp)
- **역할:** 생년월일시를 받아서 사주팔자(四柱八字)를 계산해주는 서버
- **통신 방식:** JSON-RPC (앱이 질문을 보내면 JSON 형태로 답이 옴)
- **언어:** TypeScript (Node.js 기반)

기존 GPT, DeepSeek 등 AI가 사주를 "추론"해서 틀리는 문제를 해결하기 위해 만들어졌음.
AI가 계산하는 게 아니라 **알고리즘으로 정확하게 계산**하는 것이 핵심.

---

## 1-2. 제공하는 도구 3가지

### 도구 ① getBaziDetail — 핵심 도구

**하는 일:** 생년월일시를 넣으면 사주팔자 전체 정보를 돌려줌

**입력값:**
| 파라미터 | 타입 | 설명 |
|---------|------|------|
| `solarDatetime` | 문자열 (ISO 8601) | 양력 날짜+시간. 예: `"1990-05-15T14:30:00"` |
| `lunarDatetime` | 문자열 | 음력 날짜 (양력 대신 쓸 때) |
| `gender` | 숫자 | 0 = 여자, 1 = 남자 |
| `eightCharProviderSect` | 숫자 | 자시(23~01시) 계산 방식. 학파마다 다름 |

**돌려주는 값 (반환값):**
```json
{
  "八字": "丙午 丙申 癸亥 己未",
  "日主": "癸",
  "年柱": {
    "天干": { "天干": "丙", "五行": "火", "陰陽": "陽" },
    "地支": { "地支": "午", "五行": "火", "陰陽": "陽" }
  },
  "月柱": { "天干": {...}, "地支": {...} },
  "日柱": { "天干": {...}, "地支": {...} },
  "時柱": { "天干": {...}, "地支": {...} },
  "大運": [
    { "干支": "丁酉", "开始年份": 2033, "开始年龄": 7 },
    { "干支": "戊戌", "开始年份": 2043, "开始年龄": 17 },
    ...
  ]
}
```

**정리하면 이 도구가 주는 정보:**
- 팔자 8글자 (年柱·月柱·日柱·時柱 각 天干+地支)
- 각 글자의 오행(五行)과 음양(陰陽)
- 일주(日主) — 일간(日干), 즉 태어난 날의 천간
- 대운(大運) — 10년 단위로 바뀌는 운의 흐름, 시작 나이 포함

---

### 도구 ② getSolarTimes — 역산 도구 (택일용)

**하는 일:** 원하는 사주 8글자를 넣으면, 그 사주가 나오는 날짜 목록을 돌려줌

**입력값:**
| 파라미터 | 설명 |
|---------|------|
| `bazi` | 팔자 8글자. 예: `"丙午 丙申 癸亥 己未"` (年柱 月柱 日柱 時柱 순서, 공백 구분) |

**쓰임새:** 택일(좋은 날 고르기)에서, "이런 사주를 가진 아이를 낳으려면 언제 태어나야 하나" 계산할 때 사용

---

### 도구 ③ getChineseCalendar — 황력 조회

**하는 일:** 특정 날짜의 음력 정보, 간지, 길흉 정보 조회

**입력값:**
| 파라미터 | 설명 |
|---------|------|
| `solarDatetime` | 양력 날짜. 기본값은 오늘 |

**돌려주는 값:** 음력 날짜, 일진(日辰), 길신(吉神)/흉신(凶神), 오늘 하기 좋은 일/나쁜 일 등

## 1-3. 앱이 bazi-mcp로 하는 일 (현재 로직)

```
생년월일시 입력
    ↓
getBaziDetail 호출
    ↓
8글자의 오행 개수 세기
    ↓
가장 적은 오행 = 용신으로 설정
    ↓
이름 후보 평가:
  - 발음오행: 성씨초성→이름첫자→이름둘째자 상생 여부
  - 자원오행: 한자 부수의 오행이 용신과 일치하는지
  - 수리사격: 획수 조합의 길흉 판정
    ↓
점수 합산 → 이름 추천
```

---

## 1-4. bazi-mcp가 **하지 않는** 것 (한계)

현재 이 MCP는 아래 항목들을 **계산해주지 않음:**

| 항목 | 설명 |
|------|------|
| 지장간(支藏干) | 지지 안에 숨겨진 천간들 |
| 일간 강약 판단 | 일주가 강한지 약한지 (身强/身弱) |
| 격국(格局) | 사주의 구조적 패턴 |
| 용신(用神) | 사주 균형을 잡아주는 핵심 오행 |
| 공망(空亡) | 힘이 없어지는 간지 |
| 신살(神殺) | 도화, 역마, 양인 등 특수 기운 |
| 납음오행(納音五行) | 60갑자 기반 오행 체계 |

# PART 2. bazi-mcp + 앱 로직의 문제점

## 문제 ① 용신(用神) 계산 방식이 틀림 ← 가장 치명적

**현재 방식:**
> 8글자 중 오행 개수가 가장 적은 것 = 용신

**왜 틀린가:**

용신은 단순히 "부족한 오행을 채우는 것"이 아님.
실제 명리학에서 용신 도출 순서는 다음과 같음:

```
1단계: 月令(월령) 파악
   → 태어난 달의 지지가 어떤 오행의 계절인가?
   → 예: 寅卯月 = 木의 계절, 巳午月 = 火의 계절

2단계: 일간(日主)의 강약 판단 (身强/身弱)
   → 월령에서 일간이 힘을 받는가? (得令/失令)
   → 주변 천간·지지가 일간을 생해주는가 극하는가?
   → 일간이 강하면 → 설기(洩氣)하거나 극하는 오행이 용신
   → 일간이 약하면 → 생조(生助)해주는 오행이 용신

3단계: 格局(격국) 확인
   → 월지 지장간이 천간에 투출했는가?
   → 정관격, 식신격, 재격 등 격이 무엇인가?

4단계: 용신·희신·기신·구신·한신 분류
   → 용신: 사주의 균형을 잡는 핵심 오행
   → 희신: 용신을 도와주는 오행
   → 기신: 용신을 방해하는 오행
```

**결론:** 지금처럼 단순 개수로 용신을 정하면
같은 오행 분포여도 일간이 강하냐 약하냐에 따라 용신이 정반대로 바뀔 수 있음.
→ 용신이 틀리면 이름 추천 전체가 의미 없어짐.

---

## 문제 ② 지장간(支藏干)을 무시함

**지장간이란:**
지지(地支) 12개 안에는 천간이 숨어 있음. 예:

| 지지 | 지장간 |
|------|--------|
| 子 | 壬(10%) 癸(90%) |
| 午 | 丙(10%) 己(20%) 丁(70%) |
| 寅 | 戊(10%) 丙(20%) 甲(70%) |

**문제:**
bazi-mcp는 지지의 "대표 오행"만 반환함 (午 → 火).
하지만 午 안에는 火만 있는 게 아니라 土도 있음.

현재 앱은 8글자만 카운트하는데, 실제로는 지장간까지 포함하면
오행 분포가 크게 달라질 수 있음.

**공부해야 할 것:** 12지지별 지장간 구성과 각 비율

---

## 문제 ③ 신강/신약(身强/身弱) 판단 로직 없음

용신을 제대로 내려면 일간이 강한지 약한지를 먼저 알아야 함.
이를 판단하는 기준:

- **월령득기(月令得氣):** 일간이 태어난 달의 오행에서 힘을 받는가
  - 예: 甲木 일간이 寅月(木)이나 亥月(水)에 태어나면 강함
  - 예: 甲木 일간이 申月(金)이나 酉月(金)에 태어나면 약함

- **천간의 생조:** 다른 천간들이 일간을 생해주는가

- **통근(通根):** 일간이 지지의 지장간에 뿌리를 두고 있는가

현재 앱과 bazi-mcp 모두 이 계산을 하지 않음.

---

## 문제 ④ 格局(격국) 분석 없음

격국이란 사주 전체의 구조적 패턴.
월지(月支)의 지장간이 천간에 드러났는지(透出)를 기준으로 정함.

| 격국 | 설명 |
|------|------|
| 정관격(正官格) | 일간을 극하는 음양이 다른 오행이 월지에서 투출 |
| 식신격(食神格) | 일간이 생하는 오행이 월지에서 투출 |
| 편재격(偏財格) | 일간이 극하는 같은 음양 오행이 월지에서 투출 |
| ... | |

격국에 따라 용신이 달라지고, 이름에서 강화해야 할 오행도 달라짐.

---

## 문제 ⑤ 수리사격 획수 기준 불명확

수리사격은 **강희자전(康熙字典) 획수**를 써야 하는 학파가 많음.
강희자전 획수는 현대 표준 획수와 다른 경우가 있음:

| 한자 | 현대 획수 | 강희자전 획수 |
|------|----------|-------------|
| 氣 | 10획 | 10획 (같음) |
| 海 | 10획 | 10획 (같음) |
| 草 | 9획 | 10획 (다름) |

현재 앱의 `인명용한자_DB.json`이 어느 기준인지 확인 필요.

---

## 문제 ⑥ 음양(陰陽) 배합 미고려

이름 3글자(성+이름2자)의 음양 배합도 작명에서 중요하게 봄.
bazi-mcp는 각 글자의 음양 정보를 반환하지만,
현재 앱 로직에서는 음양 배합 평가를 하지 않음.

---

## 문제 ⑦ 대운(大運)과 이름의 연관 미고려

bazi-mcp가 대운 정보를 주는데 현재 앱에서 활용하지 않음.
일부 작명 이론에서는 초년 대운의 오행을 보완하는 방향으로 이름을 지음.

# PART 3. 명리학 공부 우선순위 (작명 앱 개선 기준)

이 앱을 제대로 고치려면 아래 순서로 공부하면 효율적:

| 순서 | 공부할 내용 | 이유 |
|------|-----------|------|
| 1 | 십천간·십이지지의 오행·음양 | 모든 계산의 기초 |
| 2 | 지장간(支藏干) 12지지별 구성 | 오행 분포 정확도 |
| 3 | 오행의 旺·相·休·囚·死 (계절별 강약) | 일간 강약 판단 |
| 4 | 월령과 일간 강약 (신강/신약) | 용신 계산의 핵심 |
| 5 | 격국론(格局論) 기초 | 용신 정밀도 향상 |
| 6 | 용신·희신·기신·구신·한신 구분 | 이름 오행 방향 설정 |
| 7 | 수리사격 이론 + 강희자전 획수 | 획수 정확도 |
| 8 | 발음오행 + 자원오행 이론 | 현재 로직 검증 |

---

# 핵심 요약

```
bazi-mcp가 주는 것:
  ✅ 8글자 (천간+지지)
  ✅ 각 글자의 기본 오행
  ✅ 음양
  ✅ 대운 목록

bazi-mcp가 주지 않는 것:
  ❌ 지장간
  ❌ 일간 강약 판단
  ❌ 격국
  ❌ 용신 (앱이 직접 계산해야 함)
  ❌ 공망·신살

현재 앱의 가장 큰 오류:
  → 용신을 "오행 개수 최소값"으로 계산하는 것
  → 이걸 고치려면 신강/신약 판단 + 지장간 반영 필요
```
